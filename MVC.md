#  Spring MVC 패턴 정리

## 1. 개념

- Spring MVC는 Model, View, Controller로 역할을 나누어 개발하는 구조이다.
이 구조를 통해 코드의 유지보수성과 확장성을 높일 수 있다.

---

## 2. 구성 요소

### 1) Model
- 데이터를 처리하는 역할
- 데이터베이스와 연결되어 데이터를 저장하거나 조회

---

### 2) View
- 사용자에게 보여지는 화면
- HTML, Thymeleaf 등을 사용하여 구성한다

---

### 3) Controller
- 사용자의 요청(HTTP 요청)을 받는 역할
- 어떤 기능을 실행할지 결정한다


---

## 3. 동작 흐름

- 사용자 요청 → Controller → Service → Repository → DB → 다시 Controller → View → 사용자 출력
---

## 4. 장점

- 역할이 분리되어 유지보수가 쉽다
- 코드 구조가 명확해진다
- 협업에 유리하다

---

## 5. 정리

Spring MVC는 웹 애플리케이션을 개발할 때  
역할을 나누어 효율적으로 개발할 수 있도록 도와주는 구조이다.
